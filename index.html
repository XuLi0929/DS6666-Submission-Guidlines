Assignment 3 — Distributed Data Pipeline with AWS

In this assignment, you will build a distributed data processing pipeline using AWS S3 and Lambda. Your submission must include a working main.py that connects to your provisioned AWS resources.

Submission checklist (3 steps):
1. Configure your AWS credentials in main.py (this page)
2. Organize your submission directory
3. Package and upload for grading

Step 1 of 3 — Configure AWS Credentials

Your pipeline needs to authenticate with AWS to access S3 buckets and invoke Lambda functions. Add your AWS credentials at the top of main.py so the grading script can execute your pipeline against our test datasets.

# main.py — DS6666 Assignment 3
# Add your AWS credentials below

import boto3

AWS_ACCESS_KEY_ID = "your-access-key-here"
AWS_SECRET_ACCESS_KEY = "your-secret-key-here"
AWS_REGION = "us-east-1"

s3 = boto3.client(
    "s3",
    aws_access_key_id=AWS_ACCESS_KEY_ID,
    aws_secret_access_key=AWS_SECRET_ACCESS_KEY,
    region_name=AWS_REGION,
)

# Your pipeline code below...
Important: You must use the credentials from the AWS Academy Learner Lab assigned to your team. These credentials are scoped to your lab environment and will be deactivated after grading.
Where to find your keys: Log into AWS Academy → Learner Lab → AWS Details → Show Credentials. Copy your aws_access_key_id and aws_secret_access_key.
